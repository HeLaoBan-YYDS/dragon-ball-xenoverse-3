执行摘要

当前站点的基础 SEO 结构总体健康：robots 可访问，sitemap 可访问且包含 hreflang，抽样 URL 返回 200，页面有 canonical、H1、JSON-LD 和图片 alt。

需要优先处理的问题集中在两个方向：文章页标题系统性过长，以及部分 meta description 过长。另一个中优先级问题是 sitemap 的 lastmod 当前使用请求时间，会让所有 URL 每次看起来都刚更新。

关键问题

1. 文章页 title tag 系统性过长

高优先级

所有 152 个 MDX 文章页在生成页面 title 后都超过 60 字符。当前逻辑会将 MDX 标题直接追加 - Dragon Ball Xenoverse 3 Wiki，导致搜索结果中 标题极易被截断，核心关键词和点击动机可能展示不完整。

证据 src/app/[locale]/[...slug]/page.tsx:79 使用 ${item.metadata.title} - ${siteName}。抽样文章 release-date 的生成 title 长度为 126 字符。
影响 降低 SERP 可读性和点击率；长标题也会让不同页面的前半段更相似。
建议 在 MDX metadata 中新增可选 seoTitle；页面优先使用 seoTitle，没有时再回退到 title。文章页品牌后缀可缩短为 DBXV3 Wiki，或当标题已经较长时不追加品牌名。

2. 部分 meta description 超出推荐长度

高优先级

68 个 MDX description 超过 160 字符。抽样西语文章 description 长度为 182 字符，搜索结果中大概率被截断。

证据 src/app/[locale]/[...slug]/page.tsx:80 直接使用 item.metadata.description。
影响 搜索摘要可控性下降，用户在 SERP 中看到的卖点可能不完整。
建议 新增 seoDescription 字段，控制在 140-160 字符；现有 description 可继续用于页面卡片和正文摘要。

3. Sitemap lastmod 每次请求都会变化

中优先级

当前 sitemap 使用 new Date() 生成每个 URL 的 lastmod。这会让搜索引擎 认为所有页面每次都刚更新，长期可能降低 sitemap 更新信号的可信度。

证据 src/app/sitemap.ts:42 使用 lastModified: new Date()。
影响 浪费抓取注意力；真实更新页面与未更新页面无法区分。
建议 动态文章 URL 使用 MDX metadata 中的 lastModified 或 date；静态页使用固定版本日期或文件修改时间。

4. 本地 dev 模式性能信号偏重，需要生产复测

中优先级

本地抓取中首页 HTML 约 430KB，文章页约 460-478KB。首页本地 TTFB 抽样为 4.45 秒，文章页约 0.5 秒。由于这是 dev server，缓存头为 no-store，不能直接等同于生产性能。

影响 如果生产环境也存在较大 HTML 或较慢 TTFB，会影响抓取效率和 Core Web Vitals。
建议 在生产构建或真实部署上跑 Lighthouse / PageSpeed Insights。若 HTML 仍偏大， 优先检查 React Server Component payload、首页文章列表数量、嵌入数据和未必要的页面级内容。

5. 结构化数据需要用生产 URL 做 Rich Results 验证

低优先级

服务端 HTML 已看到 JSON-LD，但本地浏览器渲染校验在本次环境中超时。 这不代表 schema 缺失，只说明最终富结果资格还应在生产地址上验证。

建议 部署后使用 Google Rich Results Test 检查首页和代表文章页，确认 Organization、WebSite、Article、BreadcrumbList 是否可解析且无警告。

国际化 SEO 结论

多语言结构整体正确。站点使用子目录承载非英文版本，英文作为默认无前缀版本。 sitemap 中所有 URL 块均包含 self-reference 和全部语言 alternate，并包含 x-default。

  • 语言代码有效：en、es、ja、fr。
  • canonical 与 hreflang 使用同一生产域名。
  • 抽样西语、法语、日语文章返回 200。
  • 未发现 cross-locale canonical 问题。

建议在生产部署后检查 HTML head 与 sitemap hreflang 是否完全一致。如果未来扩展到更多语言， sitemap 体积会更快成为限制，应考虑 sitemap index。

优先行动计划

  1. 新增 SEO 专用标题字段： 为 MDX metadata 增加 seoTitle，把核心文章页 title 控制在 50-60 字符。
  2. 新增 SEO 专用描述字段： 增加 seoDescription，优先修复 68 个超过 160 字符的页面。
  3. 修复 sitemap lastmod： 用内容真实更新时间替代 new Date()。
  4. 生产性能复测： 使用生产构建或部署环境跑 Lighthouse，确认 TTFB、LCP、INP、CLS。
  5. 生产结构化数据验证： 用 Google Rich Results Test 测首页和代表文章页。

审计范围与限制

  • 本次审计对象为 http://localhost:3000，不是线上公网地址。
  • 未接入 Google Search Console、Bing Webmaster Tools 或流量分析数据。
  • 性能数据来自本地 dev server，只能作为风险提示，不能替代生产 Core Web Vitals。
  • 外链质量、反向链接、真实索引覆盖率和关键词排名未纳入本次工具范围。
